# KK_Crab 中控状态快照

这个目录用于把仓库外的共享控制中心运行态，回写成可提交、可同步的仓库内快照。

## 目录说明

- `current/`：最近一次同步得到的中控状态快照

## 使用方式

### 当前电脑

在离开当前电脑前执行：

```bash
tools/control-center/sync-control-state.sh
```

然后把 `.control-state/current/` 与其他代码变更一起提交、推送。

### 新电脑

拉取最新仓库后执行：

```bash
tools/control-center/bootstrap-multiwindow.sh
```

然后默认只需要打开 **1 个总控窗口**，让总控窗口基于快照恢复状态，并在内部拉起各角色 subagents。

## 注意

1. 这个目录里的内容是“可迁移快照”，不是实时共享运行态。
2. 实时共享运行态仍然在仓库外的 `<repo>-control/` 目录。
3. 如需保证换电脑后的连续性，必须先同步快照再推送。
4. 当前推荐协作方式已切换为：**总控单窗口 + 内部 Subagents + 会议落盘**。
5. 角色并没有减少，仍然保留：`前端 / 后台 / 后端 / 产品经理 / 体验设计审查 / 阻塞协调`。
6. 只有在 subagents 不可用、用户明确要求手动多窗口、或必须保留独立外部窗口时，才回退到旧的手动多窗口转发模式。
7. 若新会话需要执行 CloudBase 真实脚本，还需要把真实腾讯云密钥安全注入到**当前会话**；这部分不会随仓库快照自动恢复。
