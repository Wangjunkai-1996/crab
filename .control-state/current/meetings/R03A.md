# KK_Crab 专项会议纪要 R03A

- 会议形式：异步专项审查会
- 轮次：R03A
- 时间：2026-03-16 17:10
- 主持：总控窗口
- 参与角色：产品经理、体验设计审查

## 会议目的

在不打断三条开发线继续执行 R03 的前提下，单独发起一轮“产品 / 设计一致性专项审查”，确认当前实现方向是否与产品文档、原型和 UI 定稿一致，并提前识别必须拍板的问题。

## 审查范围

### 产品审查范围

- `docs/product/*.md`
- 当前 `board.md`、`decisions.md`、`parking-lot.md`
- 三条开发线最新 `outbox/*.md`

### 体验设计审查范围

- `docs/design/*`
- 前端 / 后台前端开发指南
- 前端、后台最新 `outbox/*.md`

## 本轮要求

1. 两个专项角色都不写代码、不改仓库文件。
2. 两个专项角色都只基于文档、会议纪要、当前回报做审查，不脑补新需求。
3. 结论必须区分：已对齐 / 不一致 / 待拍板 / 对开发线影响 / 建议总控动作。
4. 两个专项角色输出后，由总控统一收口，不直接跳过总控给开发线派需求。

## 会议后动作

1. 产品经理将结果写入 `outbox/产品经理.md`
2. 体验设计审查将结果写入 `outbox/体验设计审查.md`
3. 总控读取两个专项回报后，决定是否更新 `parking-lot.md`、`decisions.md` 与下轮 `inbox/*.md`
