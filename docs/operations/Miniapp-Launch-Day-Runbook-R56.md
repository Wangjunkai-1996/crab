## 3. 发布前检查

### T-2 ~ T-1 天

1. 对齐提审资料矩阵：名称、简介、协议、隐私、规则、截图模板、审核备注。
2. 确认发布 owner、回滚 owner、值守 owner。
3. 后端重跑 readiness / sample 检查脚本，确认恢复 gate 仍可用。
4. 后台确认治理域真实读基线与线上部署指纹无回退。

### T-60 分钟

1. 再次确认发布入口、回滚入口、证据目录。
2. 若此时安排恢复 `R54` 真实 smoke，则独立执行；失败即停止发布，不扩散到新功能排查。
3. 固定记录：
   - 发布时间
   - 发布人
   - 版本说明
   - 是否触发回滚

## 4. 回滚触发条件

满足任一项即进入回滚评估：

1. 小程序核心路径无法打开
2. `bootstrap` 持续失败且影响新用户进入
3. 报名提交 / 撤回出现稳定性失败
4. 后台治理域出现明显回退，影响上线值守处理

## 5. 值守交接物

上线前必须准备以下交接物：

1. 发布入口与回滚入口说明
2. 发布 owner / 回滚 owner / 值守 owner 清单
3. 小程序、后台、后端证据目录位置
4. 关键错误回传格式：时间、步骤、错误码、requestId、当前影响面
