- 文档目标：定义平台在通告审核、举报处理、违规处罚上的执行标准

## 2. SOP 目标

1. 保证审核尺度一致
2. 降低低质和欺诈内容流入广场
3. 保证举报和处罚有统一动作标准
4. 让冷启动阶段的人工运营可执行、可复盘

## 3. 适用范围

本 SOP 适用于以下对象：

- 新发布通告审核
- 举报通告处理
- 举报发布方处理
- 举报达人处理
- 账号处罚与黑名单管理

## 4. 审核总原则

1. 先判断内容是否真实可理解，再判断是否违规。
2. 优先保障平台安全，其次保障发布效率。
3. 对可修改的问题优先驳回，不一刀切封禁。
4. 对疑似欺诈、违法违规、恶意导流内容从严处理。
5. 所有处理动作必须记录原因。

## 5. 通告审核 SOP

### 5.1 审核前准备

审核员进入待审核列表后，按以下顺序检查：

1. 是否新用户前 3 条
2. 是否命中高风险提示
3. 标题和描述是否完整
4. 联系方式是否明显异常
5. 合作场景是否与平台定位相关

### 5.2 审核标准

#### 可直接通过

满足以下条件时可直接通过：

1. 标题和合作说明完整
2. 平台、城市、结算方式与预算信息等关键信息齐全
3. 联系方式合理
4. 无敏感违规内容
5. 无明显欺诈特征

#### 驳回待修改

出现以下情况时，建议驳回：

1. 标题过于模糊
2. 合作说明明显不完整
3. 达人要求过于空泛
4. 结算方式或预算区间无法判断
5. 联系方式缺失

#### 需补充资料

出现以下情况时，建议要求补充资料：

1. 主体信息不足以判断真实性
2. 合作内容存在较大理解歧义
3. 上传截图不清晰或关键证据缺失

#### 转人工复核

出现以下情况时，建议转人工复核：

1. 命中敏感词但语境不明确
2. 联系方式疑似异常但无法直接判定
3. 与历史违规账号存在疑似关联
4. 被多人举报后重新提交

补充说明：

- “转人工复核”应结束当前审核任务，并新建复核任务，不单独生成前台状态。

#### 直接下架或拒绝

出现以下情况时，应直接下架或拒绝：

1. 涉黄、涉赌、涉诈、违法违规
2. 明显虚假合作或诱导收费
3. 大量重复垃圾信息
4. 恶意导流到外部灰产渠道
5. 冒充品牌、门店、机构

### 5.3 审核动作模板

#### 通过模板

- 处理结果：审核通过
- 备注：内容完整，符合平台发布要求

#### 驳回模板

- 处理结果：驳回待修改
- 备注：请补充合作说明、结算方式和联系方式后重新提交

#### 补充资料模板

- 处理结果：需补充资料
- 备注：请补充主体介绍或合作证明截图，方便平台判断需求真实性

#### 拒绝模板

- 处理结果：不予通过
- 备注：内容存在明显违规或高风险，不符合平台发布规范

## 6. 举报处理 SOP

### 6.1 处理优先级

按以下顺序优先处理：

1. 涉诈、骚扰、人身安全相关举报
2. 大量重复举报的同一对象
3. 进行中的热门通告
4. 一般低质内容举报

### 6.2 举报核查步骤

1. 查看举报原因和补充说明
2. 查看被举报对象当前内容
3. 查看历史举报记录
4. 查看历史处罚记录
5. 做出处置结论

### 6.3 举报结论标准

#### 举报不成立

适用于以下情况：

1. 举报原因与内容不符
2. 内容无明显违规
3. 证据不足且历史无风险记录

#### 举报成立但轻处罚

适用于以下情况：

1. 内容低质但非恶意
2. 信息不规范但可修正
3. 首次轻微违规

对应动作：

- 警告
- 要求修改
- 临时下架

#### 举报成立且中处罚

适用于以下情况：

1. 多次重复低质发布
2. 多次被证实存在误导
3. 联系方式异常导致用户投诉

对应动作：

- 下架通告
- 限制发布 3 到 7 天
- 限制报名 3 到 7 天

#### 举报成立且重处罚

适用于以下情况：

1. 欺诈
2. 冒充主体
3. 恶意骚扰
4. 灰产导流
5. 累犯且拒不改正

对应动作：

- 全量封禁
- 加入黑名单
- 永久限制核心能力

## 7. 处罚分级规则

### 7.1 一级处罚

- 动作：警告
- 适用：首次轻微违规
- 影响：不限制核心能力

### 7.2 二级处罚

- 动作：下架通告或驳回
- 适用：低质、误导、信息严重缺失
- 影响：对应内容不可见

### 7.3 三级处罚

- 动作：限制发布或限制报名
- 适用：重复违规、被多次有效举报
- 影响：3 到 7 天内禁止核心操作

### 7.4 四级处罚

- 动作：全量封禁
- 适用：欺诈、骚扰、违法违规、严重导流
- 影响：账号进入黑名单

## 8. 典型案例判断参考

### 8.1 应通过案例

- 本地餐饮门店发布小红书探店合作，城市、预算、达人要求完整明确
- 个人 PR 发布新品试用招募，说明清晰，无诱导收费

### 8.2 应驳回案例

- 标题只有“招达人合作”
- 描述仅写“有意私聊”
- 未填写城市、平台或结算方式

### 8.3 应下架案例

- 明确要求达人先付押金
- 冒充知名品牌招聘合作
- 描述中包含违规灰产词

### 8.4 应封禁案例

- 多次被证实收取费用不履约
- 举报中存在骚扰、辱骂、诈骗证据
- 被限制后继续换号重复违规

## 9. 每日运营节奏建议

### 9.1 上午

1. 清理前一晚待审核通告
2. 优先处理高风险举报
3. 复盘昨日被驳回内容

### 9.2 下午

1. 处理新增举报
2. 复核争议内容
3. 更新黑名单和观察名单

### 9.3 晚间

1. 检查高活跃通告评论和举报情况
2. 清空高风险待处理队列

## 10. 操作留痕要求

所有审核员和运营人员必须记录：

1. 处理动作
2. 原因分类
3. 备注
4. 操作时间
5. 操作人

## 11. 升级处理机制

以下情况必须升级给运营管理员或负责人：

1. 涉及知名品牌投诉
2. 涉及法律风险
3. 涉及大规模批量违规
4. 用户强烈申诉且证据复杂

## 12. V1 执行底线

1. 不因为追求发布量而放过明显高风险内容
2. 不因为轻风控策略而放弃留痕
3. 不因为人工不足而忽略高风险举报
4. 所有封禁动作必须有明确证据和备注
