# 多米通告 状态流转表 V1

## 1. 文档信息

- 文档名称：状态流转表 V1
- 对应 PRD：[PRD-V1.md](PRD-V1.md)
- 对应审核工作流：[Review-Workflow-V1.md](Review-Workflow-V1.md)
- 文档日期：2026-03-15
- 文档目标：统一前台、后台、运营对各业务对象状态的理解

## 2. 使用原则

1. 每个对象只保留最小必要状态
2. 状态必须可追踪来源
3. 状态切换必须能映射到明确动作

## 3. 通告状态流转

### 3.1 状态定义

说明：

- “审核通过”不单独作为前台持久状态保存，而是作为后台审核动作结果，动作完成后通告直接进入“进行中”。

| 状态 | 含义 | 谁可见 | 触发人 |
| --- | --- | --- | --- |
| 草稿 | 用户已填写未提交 | 发布方 | 发布方 |
| 待审核 | 已提交等待审核 | 发布方、后台 | 发布方 |
| 驳回待修改 | 未通过，可编辑后再次提交 | 发布方、后台 | 审核员 |
| 需补充资料 | 信息不足，需补充后重提 | 发布方、后台 | 审核员 |
| 进行中 | 已审核通过并在广场展示 | 全体用户 | 审核员 / 系统 |
| 已截止 | 超过报名截止时间 | 发布方、达人、后台 | 系统 |
| 已关闭 | 发布方主动关闭 | 发布方、达人、后台 | 发布方 |
| 已下架 | 因违规或举报被下架 | 发布方、后台 | 运营 / 审核员 |

### 3.2 流转规则

| 当前状态 | 触发条件 | 下一状态 |
| --- | --- | --- |
| 草稿 | 点击提交审核 | 待审核 |
| 待审核 | 审核通过 | 进行中 |
| 待审核 | 审核驳回 | 驳回待修改 |
| 待审核 | 要求补充资料 | 需补充资料 |
| 待审核 | 转人工复核（创建复核任务） | 待审核 |
| 待审核 | 判定违规 | 已下架 |
| 驳回待修改 | 修改后重新提交 | 待审核 |
| 需补充资料 | 补充后重新提交 | 待审核 |
| 进行中 | 到达截止时间 | 已截止 |
| 进行中 | 发布方主动关闭 | 已关闭 |
| 进行中 | 举报成立或违规判定 | 已下架 |
| 已截止 | 发布方重新发布 | 待审核 |
| 已关闭 | 发布方重新发布 | 待审核 |

说明：

- “转人工复核”不会给前台新增一个“复核中”状态，通告业务状态仍保持“待审核”。

### 3.3 审核任务流转

| 任务状态 | 含义 | 触发人 |
| --- | --- | --- |
| 待处理 | 任务已进入审核队列，尚未被领取 | 系统 |
| 处理中 | 审核员已接手任务 | 审核员 |
| 已完成 | 已给出审核结果 | 审核员 / 运营 |
| 已取消 | 当前任务失效或被系统关闭 | 系统 / 运营 |

| 当前任务状态 | 触发条件 | 下一任务状态 |
| --- | --- | --- |
| 待处理 | 审核员领取 | 处理中 |
| 处理中 | 审核结果为通过 / 驳回 / 需补充资料 / 转人工复核 / 直接下架 | 已完成 |
| 处理中 | 释放任务 | 待处理 |
| 待处理 / 处理中 | 任务失效 | 已取消 |

补充说明：

1. 通告首次提交时创建“初审任务”。
2. 当审核结果为“转人工复核”时，当前任务结束，并创建新的“人工复核任务”。
3. 当通告处于“驳回待修改”或“需补充资料”后再次提交时，应创建新的“补充后复审任务”。

### 3.4 前台展示规则

1. 仅“进行中”进入广场默认展示
2. “已截止”“已关闭”“已下架”详情页不可报名
3. “驳回待修改”“需补充资料”仅发布方本人可见

## 4. 报名状态流转

### 4.1 状态定义

| 状态 | 含义 | 谁可见 | 触发人 |
| --- | --- | --- | --- |
| 已报名 | 达人已成功提交 | 达人、发布方、后台 | 达人 |
| 已查看 | 发布方已查看报名 | 达人、发布方、后台 | 发布方 |
| 待联系 | 发布方对该达人有意向 | 达人、发布方、后台 | 发布方 |
| 已沟通 | 双方进入线下或外部联系阶段 | 达人、发布方、后台 | 发布方 |
| 未入选 | 发布方确认不继续推进 | 达人、发布方、后台 | 发布方 |
| 已撤回 | 达人主动撤回报名 | 达人、发布方、后台 | 达人 |
| 已完成合作 | 合作已完成 | 达人、发布方、后台 | 发布方 |

### 4.2 流转规则

| 当前状态 | 触发条件 | 下一状态 |
| --- | --- | --- |
| 已报名 | 发布方查看 | 已查看 |
| 已报名 | 达人主动撤回 | 已撤回 |
| 已查看 | 标记有意向 | 待联系 |
| 已查看 | 标记不合适 | 未入选 |
| 待联系 | 双方已对接 | 已沟通 |
| 待联系 | 标记不合适 | 未入选 |
| 已沟通 | 合作完成 | 已完成合作 |
| 已沟通 | 放弃推进 | 未入选 |

### 4.3 限制规则

1. 同一达人对同一通告仅允许一条有效报名记录
2. “已撤回”后不可恢复原记录，只能在满足条件时重新报名
3. 通告进入“已截止”“已关闭”“已下架”后，不可产生新报名

## 5. 举报状态流转

### 5.1 状态定义

| 状态 | 含义 | 谁可见 | 触发人 |
| --- | --- | --- | --- |
| 待处理 | 举报已提交，待运营处理 | 举报人、后台 | 用户 |
| 处理中 | 运营已接手核查 | 后台 | 运营 |
| 举报成立 | 经核查确认成立 | 举报人、后台 | 运营 |
| 举报不成立 | 经核查证据不足或不符 | 举报人、后台 | 运营 |
| 已结案 | 已完成相关处罚或通知 | 举报人、后台 | 系统 / 运营 |

### 5.2 流转规则

| 当前状态 | 触发条件 | 下一状态 |
| --- | --- | --- |
| 待处理 | 运营接手 | 处理中 |
| 处理中 | 结论为成立 | 举报成立 |
| 处理中 | 结论为不成立 | 举报不成立 |
| 举报成立 | 完成处罚与通知 | 已结案 |
| 举报不成立 | 完成通知 | 已结案 |

### 5.3 联动规则

1. 举报成立可联动通告下架
2. 举报成立可联动账号处罚
3. 举报不成立不改变对象状态，但保留记录

## 6. 黑名单与限制状态流转

### 6.1 状态定义

| 状态 | 含义 | 谁可见 | 触发人 |
| --- | --- | --- | --- |
| 正常 | 无限制 | 用户、后台 | 系统默认 |
| 观察名单 | 允许使用，但提升审核等级 | 后台 | 运营 |
| 限制发布 | 暂时禁止发布通告 | 用户、后台 | 运营 |
| 限制报名 | 暂时禁止报名通告 | 用户、后台 | 运营 |
| 全量封禁 | 禁止发布和报名 | 用户、后台 | 运营 / 超管 |
| 已解除 | 处罚已结束 | 用户、后台 | 运营 |

### 6.2 流转规则

| 当前状态 | 触发条件 | 下一状态 |
| --- | --- | --- |
| 正常 | 轻度风险积累 | 观察名单 |
| 正常 | 发布违规成立 | 限制发布 |
| 正常 | 报名违规成立 | 限制报名 |
| 正常 | 严重违规 | 全量封禁 |
| 观察名单 | 风险消除 | 已解除 |
| 观察名单 | 再次违规 | 限制发布 / 限制报名 / 全量封禁 |
| 限制发布 | 到期解除 | 已解除 |
| 限制报名 | 到期解除 | 已解除 |
| 全量封禁 | 特批解除 | 已解除 |

## 7. 消息通知状态

### 7.1 状态定义

| 状态 | 含义 |
| --- | --- |
| 未读 | 用户未查看 |
| 已读 | 用户已查看 |
| 已归档 | 系统可归入历史记录 |

### 7.2 触发场景

1. 通告审核结果变化
2. 新报名产生
3. 报名状态变化
4. 举报处理结果变化
5. 处罚生效或解除

## 8. 推荐实现原则

1. 前台展示文案可以更友好，但底层状态名应稳定
2. 任何状态切换都应生成处理记录
3. 需要系统自动流转的状态应单独标注触发条件
